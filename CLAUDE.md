# CalCLI 项目规范

## 声明与禁止
- 运行命令前不需要使用`pwd`命令确认当前所在工作目录
- 不得创建`delete`命令或其他新命令
- 不得删改项目文件夹
- 所有开发必须遵循CLAUDE.md中的规范
- 有任何任务更新要写入.log/，描述任务更新的命令以及更新后任务的属性

## 操作确认流程（严格执行）
1. **部分命令操作前必须确认**：每次执行calcli命令行操作前，view和list可以不经询问地执行，其他命令必须先把完整命令和解释发给用户
2. **等待用户明确确认**：用户必须回复"确认"、"同意"或类似的明确确认语句后才能执行
3. **批量操作单独确认**：即使是批量操作，也要先展示所有命令，等待用户确认
4. **修改后重新确认**：如果用户要求修改命令，修改后必须再次展示并等待确认
5. **禁止跳过确认**：绝对不允许在未获得用户明确确认的情况下执行任何calcli命令

## 任务日志示例
**说明**：任务日志只需要记录添加了什么任务和任务属性，不要包含开发相关事宜。

```markdown
# 任务更新日志
时间: 2026-03-27 11:12:26
命令: calcli create --name "开发者测试任务和脚本思考" --description "系统性思考开发者应该如何添加测试任务和测试脚本" --begin "2026-03-27 14:00:00" --end "2026-03-27 17:00:00"

## 添加的任务属性
- 任务ID: E7363A
- 任务名称: 开发者测试任务和脚本思考
- 任务描述: 系统性思考开发者应该如何添加测试任务和测试脚本
- 开始时间: 2026-03-27 14:00:00
- 结束时间: 2026-03-27 17:00:00
- 消逝时间: 2121-02-01 21:21:00
- 重复类型: yearly (每1年)
- 创建时间: 2026-03-27 11:12:26
```
